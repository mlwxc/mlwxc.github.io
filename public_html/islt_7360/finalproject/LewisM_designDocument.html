Michael Lewis' Basic Webpage

Purpose

Goals
This website will offer a place for people to find any novel, series, or author they are looking for. This also gives users a place to browse without being limited to what their library has.
User Needs
Users would be of all ages. Able to search for a specific subgenre in a genre, such as dystopia novels in a fantasy novel. Users can find information on authors and the novels they have written. Users would be able to use a search bar for the information they want as well.
Benefits of the Website
This would become a one-stop "shop" for people to find a book. There wouldn't be the limitation of a library catalog. Users would be able to come here and find a book related to a series or novel that isn't in their library

Use Cases

  1. A student is looking for a recent book on the psychology of children and A.D.D. The subgenres would allow the user to find a book on children's psychology and under this heading they could find a book on A.D.D.
  2. The user is looking for information on a specific author. Theyare able to use the search function to find the author they are searching. The author's preferred genre of writing and the age group they write for. The page would also contain a link to the author's website, if they have one.
  3. A user is looking for information about a specific book. They are able to find the book and find novels related on the topics discussed. The book page would also contain the genres it is listed as and would link to the different pages conatining the books pertaining to that subject.

Design Concept

  1. Audience
    • Ages: I want to make this website accessible to all ages.
    • Gender: Gender is not applicable here.
    • Location: This will probably be used in the United States and other English-speaking countries. I will not be able to get this set up for all languages.
    • Living and Hobbies>: I want this to be available for all employments and hobbies. If somebody wants to find more information on birds, they can find books on the study of birds. If a physics professor wants more information on a subject, they can find it under the heading for physics and the particular portion they are curious about.
    • Computer skills: This website will be easy to navigate. The user should be able to find all of the information from the top and navigate from there if they wish.
  2. Reference Websites
    • This website is the original inspiration for my website. However, I think the navigation is clunky and the website looks extremely busy.
    • The next reference would be here. This website has a lot of good ideas, but I really like the author page. There is a list of quotes from him, the author's website, and the type of literature he writes.
    • Library catalogs are an excellent source for a search function. My local library, Daniel Boone Regional Library, uses bibliocommons to make their card catalog. The search function they use is quite amazing and this is the type of search I would like to use in my own website.
    • Project Gutenberg is a website made for novels out of copyright laws. The feature, bookshelves, is very well designed. It is separated by letter and everything is evenly spaced. This is the type of subgenre list I would like to make.
    • The last website is whichbook. This website has a slider on their homepage that allows the user to indicate what they would like more of in their book. That is impressive and I doubt I will be able to accomplish anything like this, it is something to aspire towards.

Presentation Information

  1. Content Synopsis: I want each portion to be separate. I want a thick border around the body of the website with information broken up by tables. I want the home button to be large on the very top and would prefer for it to move with the navigation through the page. If a person scrolls down the webpage I want it to follow them. This way they don't have to constantly return to the top of the website to navigate around.
  2. Text Outline: The home page has the home button/name of the website on the very top in a dashed box. Underneath will be the navigation buttons for fiction and nonfiction. Those will link to different pages containing a list of genres. Those genres would also have links to further classifications, in order, to help the user find what they need. Those classifications will be alphabetized for easy finding. The search bar would also be contained on the top of the webpage.
  3. Wireframe: Home Page Design.
  4. Rationale: Everything is clearly labeled and easy for the user to find. The navigation will be hyperlinked and separated to allow for no misclicks to happen. I am unsure of whether to put the search bar above or below the subject labels. This will be on the top for users to immediately find and use.
  5. Media: I want to put a picture of a bookcase filled with stories. Something to show the endlessness of the website. I plan to find these images on a free use website, and I will put the copyright information to the right below the picture. I also want to put images of the cover page of the book and a picture of the author on their page. Everything that is copyright will be included on the webpage.
  6. Color Scheme: The pages will have a background color of parchment with the type on the webpage being black.
